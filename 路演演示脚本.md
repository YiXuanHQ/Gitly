**话术：**

> "大家好，我是【江鸟纪元】的负责人。
>
> 今天介绍的是 **Gitly** —— 一款为 VS Code 打造的 Git 可视化助手。
>
> 我们团队有四个人，我主要负责项目的产品规划与设计和整体功能实现，剩余三人负责相关功能的实现与测试。
>
> 首先要说一下为什么我们要做 Gitly 这个 VS Code 插件。在开发过程中，我们常遇到这些问题：
>
> - Git 历史复杂，难以理解代码演进过程
> - 命令行操作繁琐，还容易出错
> - 冲突解决困难，需要手动查找和编辑
> - 缺乏直观的可视化工具
>
> **同时市场现状：** 目前 VS Code 上也有一些 Git 可视化扩展，比如 Git Graph、GitLens 等，但它们存在一些不足：
> - **语言支持不完善**：大多数插件只有英文界面，对中文用户不友好
> - **功能分散**：提交图、分支管理、冲突解决等功能分散在不同插件中
> - **缺乏智能特性**：没有命令分组、冲突可视化解决等智能化功能
> - **缺少数据分析**：无法直观了解项目活跃度和贡献者分布
>
> **Gitly 的优势：** 相比现有插件，Gitly 具有以下核心优势：
> - ✅ **完整双语支持**：VS Code 设置和 Webview 界面都完整支持中英文，这是我们的核心差异化优势
> - ✅ **三合一设计**：提交图 + 侧边栏 + 控制面板，一站式解决所有 Git 操作
> - ✅ **智能命令分组**：根据仓库状态动态显示可用命令，降低学习成本
> - ✅ **可视化冲突解决**：三栏对比视图，批量处理冲突，比传统方式更高效
> - ✅ **数据分析功能**：热力图、时间线，帮助了解项目活跃度
>
> **解决方案：** Gitly 通过**可视化提交图、智能侧边栏和功能丰富的控制面板**，让 Git 操作变得简单直观。
>
> 接下来我将从功能、技术、创新三个方面介绍 Gitly。"

## 💡 竞品对比详细说明

### 主要竞品分析

**1. Git Graph**
- 优势：提交图可视化效果好
- 不足：只有英文界面、功能单一、无智能特性
- Gitly 优势：完整双语支持、三合一设计、智能命令分组

**2. GitLens**
- 优势：功能强大、集成度高
- 不足：界面复杂、学习成本高、中文支持不完善
- Gitly 优势：界面简洁、操作直观、完整双语支持

**3. Git History**
- 优势：简单易用
- 不足：功能有限、无冲突解决、无数据分析
- Gitly 优势：功能全面、可视化冲突解决、数据分析功能

### Gitly 核心差异化优势

**1. 完整双语支持（核心优势）**
- VS Code 设置界面完整中文化
- Webview 界面完整中文化
- 这是目前市场上唯一做到完整双语支持的 Git 可视化扩展

**2. 三合一设计**
- 提交图：可视化代码演进
- 侧边栏：快速访问常用功能
- 控制面板：9 个功能模块，一站式解决

**3. 智能特性**
- 命令智能分组：根据仓库状态动态显示
- 冲突可视化解决：三栏对比，批量处理
- 数据分析：热力图、时间线，了解项目活跃度

**4. 性能优势**
- 原生实现，无框架开销
- 智能缓存，响应快速
- 支持大仓库（10万+ 提交）
